# Отчет по аудиту кода CRM-системы (NewLevel BI)

## Summary: Общая оценка здоровья кодовой базы
**Оценка: 4/10**

Кодовая база представляет собой функциональный прототип (MVP), который успешно решает базовые задачи визуализации локальных данных, но имеет серьезные архитектурные и логические ограничения, препятствующие масштабируемости и стабильности. Основные проблемы сосредоточены в ядре аналитического движка и безопасности обработки данных.

## Critical Issues: Критические баги и уязвимости
1.  **Уязвимость инъекции кода (RCE):** В файле `src/core/metrics.js` используется конструктор `new Function` для вычисления формул. Это позволяет выполнить произвольный JavaScript-код в контексте браузера пользователя при вводе злонамеренной формулы.
2.  **Нерабочая бизнес-логика BI:** Файл `src/core/explore.js` содержит заглушку вместо логики агрегации. Все графики всегда отображают только количество строк (`items.length`), игнорируя выбранные метрики (SUM, AVG, MIN, MAX) и семантические метрики.
3.  **Ошибка в логике прав доступа (RLS):** В `src/core/permissions.js` роль `viewer` (Наблюдатель) возвращает все строки данных без фильтрации, что эквивалентно правам администратора. Также отсутствует обработка неопределенных ролей (система «открывается» по умолчанию).
4.  **Отсутствие обработки тяжелых задач:** Несмотря на наличие `dataWorker.js`, он не подключен. Все операции парсинга, джойнов и профилирования выполняются в основном потоке, что приведет к «зависанию» интерфейса на файлах объемом более 10-20 МБ.

## Optimization: Рефакторинг и удаление мусора
1.  **Дублирование кода:** Функции `uid()` и `normalizeKey()` продублированы в нескольких файлах (`App.jsx`, `parser.js`, `matching.js`, `modeling.js`). Необходимо вынести их в `src/core/utils.js`.
2.  **Разгрузка App.jsx:** Компонент `App.jsx` перегружен состоянием и логикой (более 400 строк). Рекомендуется декомпозиция на кастомные хуки (например, `useWorkspace`, `useAnalytics`) и использование Context API для передачи глобального состояния.
3.  **Удаление «заглушек»:** В `src/workers/dataWorker.js` оставлен пустой плейсхолдер.

## Logic & UX: Архитектурная логика процессов
1.  **Агрессивная нормализация ключей:** `normalizeKey` удаляет пробелы, дефисы и скобки. Это может привести к коллизиям при объединении таблиц (например, «ID-1» и «ID 1» станут идентичными).
2.  **Ограниченность Semantic Layer:** Интерфейс позволяет создавать бизнес-метрики, но они никак не участвуют в расчетах `explore.js`.
3.  **Отсутствие тестирования:** В проекте нет ни одного автоматизированного теста, что критично для BI-системы, где точность расчетов является ключевым требованием.
